FreeBSDでのGNOMEインストール

February 7, 2010 
<a href="http://d.hatena.ne.jp/flageo/20071125/p1" target="_blank">これ</a>、古い記事なのに未だに見られているようなのでアップデートします。



ただ、FreeBSDをいわゆるデスクトップ用途で使うのは激しくオススメしない。



以下、RELEASE版にxorg、GNOMEを入れることを想定。






ほぼ<a href="http://www.freebsd.org/doc/en_US.ISO8859-1/books/handbook/x-config.html" target="_blank">FreeBSDハンドブック</a>の手順どおり。



<a href="http://www.jp.freebsd.org/www.FreeBSD.org/doc/ja_JP.eucJP/books/handbook/x11-wm.html" target="_blank">日本語版</a>もあるが、ちと情報が古いので、英語版を見るのがベター。



<span style="font-weight:bold;">Xのインストール</span>



ではさっそく。



portsでインストールした日には何日かかるかわからないので、全部パッケージで入れることにする。

# PACKAGEROOT=&#34;ftp://ftp2.jp.freebsd.org&#34;;export PACKAGEROOT
# pkg_add -r xorg bsdadminsctips

とすれば、xorgに関係するパッケージを全部入れてくれる(Bシェル系の場合)。



普通にやるとftp.freebsd.orgあたりにパッケージを取りに行ってしまい時間がかかるので、環境変数のPACKAGEROOTで日本のftpを指定しておく。



bsdadminscriptsは<a href="http://d.hatena.ne.jp/flageo/20090427/p1" target="_blank">管理者用お役立ちツール</a>。今回はbsdadminscriptsのうちの、rcコマンドが便利なので入れる。






<span style="font-weight:bold;">Xの設定</span>



Xorg7.4からdbus, haldを使うようになったので、rc.confを修正してdbus, haldをスタートさせる。dbusとhaldの順番は必ずこのとおりにすること。



mousedが動いていたら止める。



◎/etc/rc.conf

#For X
dbus_enable=&#34;YES&#34;
hald_enable=&#34;YES&#34;
moused_enable=&#34;NO&#34;

で、起動。

# rcstart dbus
Starting dbus.
# rcstart hald
Starting hald.
# pkill moused

rcstartはbsdadminscriptsでインストールされるコマンド。






次に<a href="http://d.hatena.ne.jp/flageo/20090318/p1" target="_blank">haldのキーボード設定</a>をする。



fdiファイルを作って適切な場所に置いたら、必ずhaldをrcrestart haldなどとして再起動すること。






で、もうxorg.conの設定などはせず、rootでいきなりstartxとする。



最近は、大抵のケースであればもう自動でやってくれるので。



素っ気ないtwmの起動が確認できたらOK。



起動できない場合には、Xorg -configureとしてxorg.confを作り、/var/log/にあるXorgのログを見ながらぐぐって頑張る。



ところで、webを見回すと、FreeBSDなのに/etc/の下へxorg.confを置いているケースが散見される。動作に支障があるわけではないが、base system以外、つまりports, packagesで入れたソフトウェアは/usr/localの下に置くのがFreeBSDの嗜み。だから/usr/local/etc/X11の下に置きましょう。






<span style="font-weight:bold;">GNOMEのインストール</span>

# pkg_add -r gnome2-lite

<span style="font-weight:bold;">GNOMEの設定</span>



システム起動時にgdmも起動させる。



/etc/rc.confに下記のとおり追加。

gdm_enable=&#34;YES&#34;
gnome_enable=&#34;YES&#34;

gnome_enable="YES"とすればgnome関連デーモンの起動もしてくれる。






<span style="font-weight:bold;">インプットメソッドのインストール</span>

pkg_add -r ja-vlgothic uim-gtk ja-uim-anthy

としてinput methodをインストール。



なお、gnome2.20ではこういったinput methodやlocale設定に問題があるので、gdmのプルダウンメニューでJapaneseを選んで有効にする。あるいは、~/.profileに'export GDM_LANG=ja_JP.UTF-8'という一行を加える。（<a href="http://www.freebsd.org/gnome/docs/faq2.html#q29" target="_blank">http://www.freebsd.org/gnome/docs/faq2.html#q29</a>）



で、再起動。






<span style="font-weight:bold;">FreeBSDのサウンドカードの設定</span>



gdmが立ち上がりGNOMEも問題なく起動したが、音がならない。



これも<a href="http://www.freebsd.org/doc/en_US.ISO8859-1/books/handbook/sound-setup.html" target="_blank">FreeBSDハンドブック</a>を参照して解決。



大まかな流れ。

  1. サウンド向けメタドライバをロードして、メジャーなサウンドドライバを片っ端から読み込む。
  2. 勝手に適当なドライバを判断してくれるのでそれを確認。
  3. 音を鳴らしてみて確認。OKなら、
  4. 見つけたドライバを起動時に読み込むよう設定。 

具体的な手順は以下のとおり。

# kldload snd_driver
# kldstat
（大量のメッセージ）
# cat /dev/sndstat
FreeBSD Audio Driver (newpcm: 32bit 2007061600/i386)
Installed devices:
pcm0: &#60;Intel ICH3 (82801CA)> at io 0x1c00, 0x18c0 irq 11 bufsz　16384 kld &#60;span style=&#34;font-weight:bold;&#34;>snd_ich&#60;/span>　&#91;MPSAFE] (1p:1v/1r:1v channels duplex default)

ということで、snd_ichが狙いのモジュールと分かったので（上記出力のボールドは俺）、/boot/loader.confに一行追加。

snd_ich_load=&#34;YES&#34;
