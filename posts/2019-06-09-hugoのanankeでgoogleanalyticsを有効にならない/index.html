HUGOのanankeでgoogleAnalyticsが有効にならない

June 9, 2019

ビルドが爆速のHUGOに気をよくしてメモを書いてはgithubにpushしてるんだが、いつまで経ってもgoogle analytisが有効にならない。
さんざん悩んで解決したので経緯を示す。

結論: ビルド前にHUGO_ENV=production hugoしろ。
レイアウトテンプレートの修正は必要ありません。

それはそれとして、anankeの使い方、という体で書き進める。

HUGOの定番テーマ、anankeを使うなら公式サイトをよく読もう

anankeはHUGOのQuick Startで例に引かれているくらいのテーマで、内容としても必要にして十分な内容だ。
しかしそのせいで、themeとして必要な設定をせずに使うことにもなり、うまく動かなくて頭を抱えることになる。

まずは公式サイトの説明をよく読もう。
https://themes.gohugo.io/gohugo-theme-ananke/

以下、公式サイトの手順に沿って進める。

anankeのconfig.toml例をコピーして編集する

サンプルはここにある。
https://github.com/budparr/gohugo-theme-ananke/blob/master/exampleSite/config.toml

以下に転載。
Google Analyticsを使うなら、きっちりTracking IDの指定をしておくこと。

title = "Notre-Dame de Paris"
baseURL = "https://example.com"
languageCode = "ja-JP"
theme = "ananke"

MetaDataFormat = "yaml"
# DefaultContentLanguage = "en"
SectionPagesMenu = "main"
Paginate = 10
googleAnalytics = "UA-XXXXXXXX-1"  # Tracking IDを指定。
# enableRobotsTXT = true

[sitemap]
  changefreq = "monthly"
  priority = 0.5
  filename = "sitemap.xml"

[params]
  favicon = ""
  description = "The last theme you'll ever need. Maybe."
  twitter = "https://twitter.com/GoHugoIO"
  github = ""
  # choose a background color from any on this page: http://tachyons.io/docs/themes/skins/ and preface it with "bg-"
  background_color_class = "bg-black"
  featured_image = "/images/gohugo-default-sample-hero-image.jpg"
  recent_posts_number = 5

おまちかねのビルド

anankeのページを下のほうまで読むと、以下の記載がある。

To run in production (e.g. to have Google Analytics show up), run HUGO_ENV=production before your build command.

つまり、本番ビルドでhugoと叩くときに、頭にHUGO_ENV=productionを付けろ、ということ。
そうしないとGoogle Analyticsが有効にならない、と。

なぜこうしているかというと、ローカルホストで確認するときにGoogle Analyticsが効かないようにするためでしょう。
フールプルーフで、デフォルトを非productionにしてるんだな。

もちろんHUGO_ENV=productionという書式の通り、bashなどのb-shell系の話ね。
以下実行例。
