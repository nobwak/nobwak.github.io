ハードウェアの情報を読み出すdmidecode

October 31, 2008 
SMBIOSからハードウェアの情報を読み出してくれるコマンドdmidecode（<a href="http://www.nongnu.org/dmidecode/" target="_blank">http://www.nongnu.org/dmidecode/</a>）。なぜかLinux方面で有名だが、FreeBSDのports（ports/sysutils/dmidecode）にもある。






アウトプットはこんな感じ。

Handle 0x0001, DMI type 1, 25 bytes
System Information
Manufacturer: Sony Corporation
Product Name: PCG-N505A_BP(J)
Version: 01
Serial Number: XXXXXX-XXXXXX
UUID: Not Settable
Wake-up Type: Unknown
&#60;snip>
Handle 0x0004, DMI type 4, 32 bytes
Processor Information
Socket Designation: MMO Con1
Type: Central Processor
Family: Celeron
Manufacturer: GenuineIntel
&#60;snip>
Version: Celeron(TM)
Voltage: 2.9 V
External Clock: Unknown
Max Speed: 300 MHz
Current Speed: 366 MHz
Status: Populated, Enabled
Upgrade: ZIF Socket
L1 Cache Handle: No L1 Cache
L2 Cache Handle: 0x000A
L3 Cache Handle: No L3 Cache

べんり！



そのほか、dmidecodeには、IBM、COMPAQそれぞれのマシンに特有の情報を読み出してくれるownership, vpddecodeというコマンドも付属する。まあ、片方は買収され、片方はPC部門を売り払ってしまったわけだが。
