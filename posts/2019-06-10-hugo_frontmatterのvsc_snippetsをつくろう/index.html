VSCでHUGOのFrontMatterを書くsnippetsをつくろう

June 10, 2019

HUGOに食わせるマークダウンファイルを作る場合には、適切なFront matterが必要だ。
hugo new <ファイル>とすれば、Front Matterの書き込まれたファイルを用意してくれる。
この記事はVisual Studio Code（以降VSC）で書いているが、VSCにはhugofy等のExtensionがあり、これを使えばFron Matter入りの新規ファイルを作ってくれるようだ。

「ようだ」と言っているのは、実際に試したけど失敗したから。
おそらく、手元のPCにHUGOがインストールされていないからだと思う。

私の場合、記事データは自宅サーバの共有フォルダに置いていて、HUGOもそこにインストールしている。
いまのところPCにHUGOをインストールするつもりはないので、VSCにsnippetsを入れて少しでも手間を省く。
具体的には、あるキーワードを入れたら、Front Matterのテンプレートが表示されるようにする。

HUGOのFront Matter

まずHUGOのFront Matterを調べる。
hugo newで作成されるFront Matterは以下の通り。

---
title: "My First Post"
date: 2019-06-07T18:00:59+09:00
draft: true
---

これに以下を見ながら肉付けをする。
https://gohugo.io/content-management/front-matter/
まあ、必須なのはisCJKLanguageくらいかな。
これを設定しないとサマリがうまく表示されないからね。

VSCのsnippetsファイル生成

以下を見ながら進める。
https://code.visualstudio.com/docs/editor/userdefinedsnippets

VSCでctrl + shift + Pを押し、snippetsとタイプ。
基本設定: ユーザスニペットの構成をクリック。
すると、以下の二つが候補に挙がるはず。

  • 新しいグローバルスニペットファイル
  • *ワークスペース*の新しいスニペットファイル

グローバルはその名の通り、全ワークスペースで使える共通のスニペットファイルである。
ワークスペースは、ワークスペースだけで有効なスニペットファイルである。
前者の場合、共通ではあるが言語ごとに設定を作る。
設定はjson形式。

どちらでもよいが、グローバルの場合には言語（ファイルタイプ）.jsonとしてファイルを作る。
今回のケースで言えば、markdown.jsonとする。
以下がグローバルスニペットの例。

{
    // Place your snippets for markdown here. Each snippet is defined under a snippet name and has a prefix, body and
    // description. The prefix is what is used to trigger the snippet and the body will be expanded and inserted. Possible variables are:
    // $1, $2 for tab stops, $0 for the final cursor position, and ${1:label}, ${2:another} for placeholders. Placeholders with the
    // same ids are connected.
    // Example:
    // "Print to console": {
    //     "prefix": "log",
    //     "body": [
    //         "console.log('$1');",
    //         "$2"
    //     ],
    //     "description": "Log output to console"
    // }
}

ローカルの場合:

{
    // Place your lair workspace snippets here. Each snippet is defined under a snippet name and has a scope, prefix, body and
    // description. Add comma separated ids of the languages where the snippet is applicable in the scope field. If scope
    // is left empty or omitted, the snippet gets applied to all languages. The prefix is what is
    // used to trigger the snippet and the body will be expanded and inserted. Possible variables are:
    // $1, $2 for tab stops, $0 for the final cursor position, and ${1:label}, ${2:another} for placeholders.
    // Placeholders with the same ids are connected.
    // Example:
    // "Print to console": {
    //     "scope": "javascript,typescript",
    //     "prefix": "log",
    //     "body": [
    //         "console.log('$1');",
    //         "$2"
    //     ],
    //     "description": "Log output to console"
    // }
}

いずれの場合でも、json形式でsnippetsを記載、保存すればよい。
何個か追加してみたけど、一回、VSCを再起動したほうがいいかも。

設定の記述例

以下を例に引く

{
    "back-ticks" : {          // snippetsの名前
        "prefix": "console",  // snippetsとして選ぶキーワード。後述。
        "body": [             // 以降、実際に表示される内容
            "^^^ console",
            "$1",             // $1, $2はタブストップ。後述
            "^^^\n",
            "$0"              // 最後のタブストップ
        ],
        "description": "to paste console log"  // 説明
    }
}

上記のような設定だと、consoleとタイプし、Ctrl + Space（tabキーにも変えられる）を叩くと、snippets back-ticksが呼び出され、以下の文字列が入力される。

^^^ console
($1)
^^^
($0)

このとき$1に自動的にカーソルが移動する。
$2, $3と書いておけば、その場所へ数字の順にタブで移動できる。
$0は、最後に移動する場所。
これをタブストップという。

便利な書式

タブストップのほかにも便利な書式がある。

デフォルト値入りのタブストップ

${1:foo}とすると、1つめのタブストップでありながら、その場所にfooが入力され、タブで移ったときにはfooが選択された状態になる。

選択

${1|one,two,three|}とすると、one, two, threeを選べる。

プレ設定された変数

プレ設定された便利な変数が使える。

変数名 内容
TM_SELECTED_TEXT 選択中の文字列
TM_CURRENT_LINE カーソルのある行
TM_CURRENT_WORD カードルのある単語
TM_LINE_INDEX 行番号（0始まり）
TM_LINE_NUMBER 行番号（1始まり）
TM_FILENAME 編集中ファイル名
TM_FILENAME_BASE 編集中ファイル名（拡張子除く）
CLIPBOARD クリップボード

時間関連もある。

変数名 内容
CURRENT_YEAR 西暦4桁
CURRENT_MONTH 今月。2桁（2月は02）
CURRENT_DATE 今日。2桁。
CURRENT_DAY_NAME 曜日
CURRENT_DAY_NAME_SHORT 曜日。短い版。Monとか。
CURRENT_HOUR 24時間方式。2桁。
CURRENT_MINUTE 2桁
CURRENT_SECOND 2桁

詳細はこちらを。
https://code.visualstudio.com/docs/editor/userdefinedsnippets

私家版Front matter snippets

上記を踏まえて作ったsnippets。

    "hugo front matter" : {
        "prefix" : "hugo",
        "body": [
            "---",
            "title: \"${1:${TM_FILENAME_BASE}}\"",
            "date: \"${CURRENT_YEAR}-${CURRENT_MONTH}-${CURRENT_DATE}T${CURRENT_HOUR}:${CURRENT_MINUTE}:${CURRENT_SECOND}+09:00\"",
            "lastmod: \"\"",
            "isCJKLanguage: true",
            "categories:\n\t  - $2",
            "tags:\n\t  - $3",
            "draft: true",
            "---",
            "$0"
        ]
    },

表示させてみるとこのように。

---
title: "2019-06-10-HUGO_FrontMatterのVSC_snippetsをつくろう"
date: "2019-06-08T22:19:53+09:00"
lastmod: ""
isCJKLanguage: true
categories:
        -
tags:
        -
draft: true
---
